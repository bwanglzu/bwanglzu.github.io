Data Deluge -- Getting And Cleaning Data

Goal of getting and cleaning data: Raw data -> Processing Script -> Tidy data -> Data analysis -> Data communication.

Data are values of qualitative of quantitative variables, belonging to a set of items.

Raw data:

  1. The original source of data.
  2. Hard to use for data analysis.
  3. Only need to be processed once.

Processed data include merging, subsetting, transforming, etc, all processed data should be recorded.

Tidy data:

  1. Each variable in one column.
  2. Each ovservation in different column.
  3. Each kind of variable in one table.

Variable names should be readable.

Data Source

Downloading files

In RStudio, use getwd() or setwd() to get/set(relative or absoulte directory) your working directory.

Check if the directory exists with

if(!file.exists("data")){
    dir.create("data")
}

Download file:

fileURL <- "https://data-file-url-example.csv"
download.file(fileURL, destfile = "./data/res.csv", method = "curl")
dateDownloaded <- date()

Note: If we use Mac and the fileURL starts with HTTPS, we need to set the third argument as "curl".

Flat File

Define flat file:

A flat file database is a database which, when not being used, is stored on its host computer system as an ordinary, non-indexed "flat" file. To access the structure of the data and manipulate it, the file must be read in its entirety into the computer's memory.

use read.table() to load flat files into R, stored in RAM.

Note: read.table() is not the best way to read large data sets.

  • Excel files, most widely used format for sharing data.

Note: read.xlsx function need to install xlsx package, rJava package and Java SE 6.

library(xlsx)
fileUrl <- "https://excel-file-url"
download.file(fileUrl, destfile = "./data/res.xlsx", method = "curl")
dateDownloaded <- date()

We use read.xlsx to read xlsx file data into R.

res <- read.xlsx("./data/res.xlsx", sheetIndex = 1, header = T)

Also, we can subset data with two arguments: colIndex and rowIndex:

res <- read.xlsx("./data/res.xlsx", sheetIndex = 1, colIndex = 1:2, rowIndex = 2:4)

Means get the data of row 2,3,4, column 1,2 from sheet 1 of res.xlsx.

Note: read.xlsx2 is faster than read.xlsx but unstable than read.xlsx when subsetting data.

install.packages("XML")
library("XML")
xmlUrl <- "http://xml-file-path"

Access the child noode:

rootNode[[1]]
rootNode[[3]][[1]]//get the 1st child of node 3 under the root node

xmlSApply can help us extract parts of the xml file:

xmlSApply(rootNode[[3]], xmlValue)

rootNode[[3]] will return all student items and xmlSApply function will apply xmlValue to every student node, which returns raw content of each student node.

Reading Data

From MySQL

First of all

install.packages("RMySQL")
library("RMySQL")

Connect/Disconnect:

myDB <- dbConnect(MySQL(), user="root", password = "password", host="127.0.0.1")
res <- dbGetQuery(myDB, "show databases;")
dbDisconnect(myDB)
res

We can also add a "db" parameter to decide which databse to access:

mydb <- dbConnect(MySQL(), user = "root", password = "password",db = "ipmapping",host="127.0.0.1")
tables <- dbListTables(mydb)
fields <- dbListFields(mydb, "ipbelong")
query <- dbGetQuery(mydb, "select count(*) from ipbelong")
data <- dbReadTable(mydb, "ipbelong")

dbSendQuery commend allow you send a query to databse without data return, is's useful cuz we can subset result to save memory.

query <- dbSendQuery(mydb, "select * from ipmapping;")
res <- fetch(query, n = 10)

After fetch the data to localhost, use dbClearResult commend to clear the query on the remote server:

dbClearResult(query)
dbDisconnect(mydb)

Notice: only select, do not insert, delete or update.

From APIs

API = Application programming interfaces.

Accessing data from an website with oauth, we need to regist an applications on this website, then:

app = oauth_app("weibo", key = "key", secret = "secret")
sig = sign_oauth1.0(app, token = "token", token_secret ="tokensecret")
data = GET("https://api.somewebsite", sig)

Understand OAuth 2.0

Components: Resource Owner(User), Third Party Application(Client), Authorization Server, Resource Server.

Client need to get User Data from API Provider on Resource Server. Process:

  1. Client request authorization.
  2. User grant authorization.
  3. Client contact Authorization Server.
  4. Authorization Server return token to client.
  5. Client access Resource Server with token.
  6. Resource Server return data to client.

Subsetting and Sorting

subset data sets with row/col number or row/col name:

data[,1]//return col 1
data[,"col1"]
data[1:2, "col1"]//get the first two rows elements of col1

Subset with logical staments:

x[x$col1 <=3,]//get all cols where the first col value less than 1
x[(x$col1 <= 3 & x$col2 >4),]
x[(x$col1 <= 3 | x$col2 >4),]

If the data has NA, for example:

col1 col2
111  NA
222  NA
1324 9

Subset:

x[x$col2 > 8]

It'll return the result together with NA. Use which function to ignore NA values:

x[which(x$var2>8),]

Sorting with function sort:

sort(data$col1)
sort(data$col1, decreasing = TRUE)

If the column contains NA value, sort will automatically drop these NAs. We can use na.last argument to keep these value:

sort(x$col2, na.last = T)

Order a data set with particular variable:

x[order(x$col1),]

If col1 has some same variables, we set another col to order the dataset:

x[order(x$col1, x$col2),]

Sorting with plyr package:

install.packages("plyr")
library(plyr)
arrange(dataframe, col1)
arrange(dataframe, desc(col1))

Add new col to a dataframe:

x$col4 <- c(1,2,3,4,5)
cbing(x, c(1,2,3,4,5))

Summarzing Data

Use head, tail function to look a bit of the data:

head(dataframe, 5)
tail(dataframe, 5)

Use summary to overall summary of data:

summary(dataframe)

str function for more depth information:

str(dataframe)

quantile function to view data variablity:

quantile(dataframe$colname, na.rm = T)
quantile(dataframe$colname, probs = c(0.5, 0.7, 0.9))

table is use table to show variables distribution:

table(data$col, useNA= 'ifany')

The argument useNA = 'ifany' will add na column if the dataset contains na values.

Initial a 2-dimension table with:

table(data$col1, data$col2)

Use is.na the check for missing values:

sum(is.na(data$col))
any(is.na(data$col))

To check if data meet the condition:

all(data$col > 5)

To check rows/columns sum number in dataset:

colSums(is.na(data))
rowSums(is.na(data))

Check values with specific characteristics:

table(data$col in c("1","2"))

will return the sum of column data which are equal to 1 or 2. Return these data:

data[data$col in c("1","2")]

Use object.size to check the size of dataset:

object.size(data)
print(object.size(data), units = "Mb")

Create new variables

create index for dataset:

seq(1, 10, by = 1)

x <- c(1,21,33,2,41,42,11)

data$column %in% c("data1", "data2")

resData <- ifelse(data$col < 0, TRUE, FALSE)
table(resData)

This function set a condition data$col < 0, if element meet this condition, will return TRUE, else return FALSE.

Reshaping data

install.packages("reshape2")
library(reshape2)

Melt(糅合) data frame:

melt(data, id = c("col1", "col2"), measure.vars = c("col3","col4"))

This function will melt col3 and col4 together and show together with col1 and col2, the original data frame was transformed to have more cols but less rows.

Cast data frames into different shapes:

dcast(data, col ~ variableCol, function)

This function will caregory col column and apply function to these caregorized data.

split function can split data according columns:

split(InsectSpray$count, InsectSpray$spray)

dplyr package

Is a package about working with data frames. Use mtcars data set as an example:

Select:

library(dplyr)
packageVersion("dplyr")
select(mtcars, mpg:disp)

This will return columns from mpg to disp. Add a minus will return the other columns:

select(mtcats, -(mpg:disp))

match function returns the index of column:

gearIndex <- match("gear", names(mtcats))

filter function is used to subset rwos based on conditions.

hp.f <- filter(mtcars, hp > 175)
hp.f <- filter(mtcars, hp > 175 & gear > 4)
hp.f <- filter(mtcars, !is.na(gear))

arrange function arrange column data from lowest to highest.

gear.arrange <- arrange(mtcars, gear)
gear.arrange <- arrange(mtcars, desc(gear))

rename function allow us to rename a column name:

data <- rename(mtcars, waitingtime = wt)

The left argument of expression 'waitingtime = wt' is the new column name, and the right one is the old one.

mutate function is to create or transform variables:

data <- mutate(mtcars, wtDetrend = wt-mean(wt, na.rm = T))

This function will create a column named wtDetrend.

data <- mutate(mtcars, gearNum = factor(1*(gear>4), label = c("v4", "v6")))

Tip: tbl_df function cann make data frame more compact and readable.

group_by function allow us to group data by column name:

group_by(mtcars, gear)
summarise(group_by(mtcats,gear), mean(wt))

This will return the mean value of wt group by gear.

Merging data

Merging two data frames are based on the common columns.

merge(dataframe1, dataframe2, by.x="col1Common", by.y = "col2Common", all = TRUE)

This function will merge datafram1 and dataframe2, which by.x&by.y function defined their common column name separately.

Note, plyr package also provide join function to merge dataframes, join_all can merge three or more datasets.

Dates

d1 <- date() //Character class
d2 <- Sys.Date() //Date class

format function is used to formatting dates:

  1. %d = 0-31
  2. %a = abbreviated weekday
  3. %A = unabbreviated weekday
  4. %m = 0-12
  5. %b = abbreviated month
  6. %B = unabbreviated month
  7. %y = 2 digit year
  8. %Y = 4 digit year
format(d2, "%a %b %d")
as.Date(d2, "%a %b %d")

Julian date time:

weekdays(d2)
months(d2)
julian(d2) // the number of days since 1970-1-1

Use Sys.timezone to get current timezone:

Sys.timezone(location = T)
