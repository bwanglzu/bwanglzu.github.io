Basic

get working directory:

getwd()

R can be used as a calculator, type:

4 + 9

Returns 13.

Create a variable, x gets 1 plus 3:

x <- 1 + 3
x

Returns 4.

Create a vector with c(value,value...):

c(1, 3.14, 6.28, 9)

Of course, assign the vector to the variable:

z <- c(1, 3.14, 6.28, 9)

Vectors can be combined together:

c(z, 250)

Returns: 1 3,14 6.28 9 250.

Also, we can operate vectors:

z * 2

Returns 2 6.28 12.56 18.

Operators include +, -, /, ^, sqrt(), abs().

When given two vectors the same length, vector + vector means elements do the operate element-by-element. Else, R reuse the short vector's elements. So

z * 2

means

z * c(2, 2, 2, 2)

The length of the longer vector divide the length of the shorter vercor need to be a integer, else it will raise an warning.

If forgot the variable name, just type the first two letters and use Tab key.

Missing Values

NA is used to represent not available values or missing values.

x <- c(1, NA, 2)
y <- x * 3
y

Returns 3 NA 6. Test whether an element is NA use is.na() function:

x <- c(1, 2, 3, NA, 4, NA, 5)
naVlaue <- is.na(x)
x[!naValue]

Remove NA values from multiple dataframes:

goodValue <- complete.cases(dataframe)
dataframe[goodValue, ]

Returns False True False. Use sum(my_na) return the number of True.

NaN is another type of missing value, means 'not a number'.

0/0
Inf - Inf

Both of these expressions returns NaN, Inf in R stands for infinity. Test it's class use is.nan().

A NaN value is also NA but the converse is not true.

Setting Vectors

Extract elements from a vector base on conditions.

Returns 1 1 1 1 1. Another Example:

my_vec <- c(1,2,NA,3,NA,5)
my_vec[is.na(my_vec)]

Returns NA NA. !is.na(my_vec) returns 1,2,5.

my_vec <- c(1,2,NA,-3,NA,5,-1)
my_vec[my_vec>0]

Returns 1 2 NA NA 5, as NA is a placeholder and NA>0 returns NA. If we need to get a solid vector of positive numbers:

my_vec <- c(1,2,NA,-3,NA,5,-1)
my_vec_new <- my_vec[!is.na(my_vec)]
my_vec_new[my_vec_new>0]

Returns 1 2 5, A more simple way:

my_vec[!is.na(my_vec) & my_vec>0]

R use one-based index, the first element is 1, not 0!!

Logic

Logical operations in R. Logical values equals Boolean values, are TRUE and FALSE.

Logical operations require logical operators.

'==', equality operator, '<','<=','>','>=','!=','!','and','or'.

The difference between '&' and '&&':

TRUE & c(TRUE, FALSE, TRUE)
TRUE && c(TRUE, FALSE, TRUE)

The first line returns TRUE FALSE TRUE, The second line returns TRUE. "&&" operator only compare the first element in the right vector.

'|' and '||' follow a similar set of rules.

And operators evaluated before or operators.

5 == 5.0

Returns TRUE. use the function isTRUE to check whether the expression is true.

isTRUE(1 < 2)

Also, identical()function to judge the equality of 2 arguments passed in. xor() just the same, stands for exclusive OR.

True ⊕ False = True
True ⊕ True = False
False ⊕ True = True
False ⊕ False = False

Use which(vector) to find the indices of ints match specific condition.

vect <- c(1,2,3,4,5)
which(vect > 4)

Returns 5. any() function returns TRUE if one or more elements is true, all() function returns TRUE if all elements are true.

vect <- c(1,2,3,4,5)
any(vect > 5)
all(vect >= 1)

Returns FALSE TRUE.

Functions

Functions in R are small pieces of reusable code, such as:

Sys.Date()

Mean() function calculate the mean value of a vector:

mean(c(1,3,5))

Define a user function:

my_function <- function(x){
    x
}

Function my_function will return x. View source code of function just by type it's name.

We can set a default value to the argument.

my_function <- function(x = 1){
    x
}

Pass function as arguments in another function:

calculate <- function(func, myvect){
    func(myvect)
}
calculate(sd, c(1,3,5,7,9))
calculate(mean, c(1,3,5,7,9))
calculate(median, c(1,3,5,7,9))

Three returned values represent standard deviation, mean value and median value respectively.

Use anonymous function as an argument:

calculate <- function(func, myvect){
    func(myvect)
}
calculate(function(x){x + 1}, 6)

Returns 7.

Ellipses can be used to pass on arguments to other functions that are used within used-defined function:

connectstring <- function(...){
    paste("BEGIN",...,"END", sep=" ")
}

Sometimes we need to 'unpack' the list arguments of '...', an example:

add_alpha_and_beta <- function(...){
    args <- list(...)
    alpha <- args[['alpha']]
    beta <- args[['beta']]
    alpha + beta
}

User-define-binary-operator:

"%mult_add_one%" <- function(left, right){
    left * right + 1
}
4%mult_add_one%5

Returns 21.

... argument is used when the number of arguments passed into the function can not known in advance, such as paste.

args(paste)

The sequence of search a function is:

global environment -> stats package..... ->base package

if user import a library, the sequence change to:

global environment -> namespace of the library -> stats pachkage... -> base package

R have separate namespace for functions and non-functions, so the object and a function name could be the same.

Scoping Rules(函数的作用域), R use lexical scoping(static scoping) and dynamic scoping.

f <- function(x, y){
    x^2 + y / z
}

In the function above, x and y are formal arguments, and z is free variables(free variables are not formal arguments and are not defined in the function body whichi are usually called local variables). Scoping rules define how values are assigned to free variables.

Lexical scoping search the value of free variables from the environment where the function is defiened.

environment is a set of key-value pairs. Except empty environment, every environment has it's parent environment.

The process of search the value of free variable:

  1. search in the environment where the function is defined.
  2. search it's parent environment until hit the top-level environment(global environment or package library namespace).
  3. Search the sequence of search list to base package.
  4. until to the empty environment, throw an error.

In R, it's possible to define a function in another function, and return a function.

Apply

Is used to evaluate a function to rows/cols of a matrix.

x <- matrix(rnorm(60),20,3)
apply(x, 1, mean)//apply mean to 20 rows
apply(x, 2, mean)//apply mean to 3 cols

for sums and means, have some shortcuts

rowSums = apply(x, 1, sum)
rowMeans = apply(x, 1, mean)
colSums = apply(x, 2, sum)
colMeans = apply(x, 2, mean)

These for functions are much more efficient. The forth argument is ..., means arguments of the function:

apply(x, 1, quantile, probs = c(0.25, 0.75))

(四分位数，Q1和Q3).

Lapply & Sapply

Two most important function in R (loop functions).

Lappy, Sapply, vapply and tapply offer a contribute to data analysis strategy named Spilt-Apply-Combine.

Spilt-Apply-Combine

Use head(dataset) to view the first 6 lines of dataset, and dim(dataset) returns the count of row/column nuber. To preview it's class, use 

class(dataset)

Lapply takes a list as input and apply the second argument function to each element.

mylist <- lapply(dataset, func)

Return a list, and lapply means list. List contains mutiple forms of data, transform them to character with

mylist <- lapply(dataset,func)
as.character(mylist)

And this process can be simplified by sapply:

mylist <- sapply(dataset,func)

Returns a vector contains character elements. The range() function returns the minimum and maximum of its first argument, unique() function returns a vector with all duplicate elements removed.

vapply and tapply

Vapply allows us to specify the result of data format, if the result dosen't match what you expected, it'll raise an error.

vapply(dataset, unique, numeric(1))

If the result equals our expectation, vapply returns the same as sapply.

Compare vapply and sapply:

  • vapply is safer since it allows you to specify the format type.
  • vapply perform faster than sapply for large datasets.
  • sapply saves some arguments for typing time and good enough in most cases.

Tapply allows u to spilt data up into groups based on variable values and apply a function to the members of each group.

tapply(data, groups, func)

That means apply function func to dataset data group by groups.

Mapply&tapply

Is a function contains multi variables, for example, lapply takes a list as variable then mapply can takes two.

list(rep(1, 4), rep(2, 3), rep(3, 2), rep(4, 1))
equals
mapply(rep, 1:4, 4:1)

In this mapply, the rep function was applied on 1, 2, 3, 4 and the second argument is 4, 3, 2, 1, means 1 for 4 times, 2 for 3 times...

tapply is used to apply function over subsets of a vector.

Create a variable with 3 groups, a variable with the same length with three levels, and apply mean function to data:

x <- c(rnorm(10), runif(10), rnorm(10,1))
f <- gl(3, 10)
tapply(x, f, mean, simplify = TRUE)

split function takes a vector/list/dataframe as data and a factor as levels, split function will split data into groups. Once splited, we can use apply/sapply/lapply to these individual groups.

x <- c(rnorm(10), runif(10), rnorm(10,1))
f <- gl(3, 10)
split(x, f, mean, simplify = TRUE)

Simulate

R's one adventage is to simulate random numbers.

sample(1:6, 4, replace = TRUE)

This function allow user to select 4 random numbers among 1 to 6, replace = true means the number used before was put back to the data pool. replace arguments' default value is FALSE.

LETTERS is predefined in R represents a-z. sample() function can be used to permute these elements.

set.seed allows us to reproduce random numbers.

设定随机数种子，一个特定的种子可以产生一个特定的伪随机序列，这个函数的主要目的，是让你的模拟能够可重复出现，因为很多时候我们需要取随机数，但这段代码再跑一次的时候，结果就不一样了，如果需要重复出现同样的模拟结果的话，就可以用set.seed()。在调试程序或者做展示的时候，结果的可重复性是很重要的，所以随机数种子也就很有必要。
